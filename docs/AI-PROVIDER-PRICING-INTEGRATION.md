## 🛠️ 技术实现

### 1. 文档结构设计

#### 供应商信息架构

```json
{
  "name": "供应商名称",
  "baseUrl": "API基础URL",
  "models": ["支持的模型列表"],
  "description": "供应商特色描述",
  "pricingUrl": "价格页面链接"
}
```

#### 文档组织方式

- **分组展示**: 按国际/国内供应商分类
- **信息完整性**: 包含Base URL、模型列表、特点描述、价格链接
- **易读性**: 使用表格和图标增强可读性
- **可维护性**: 统一的数据结构便于更新

### 2. 脚本工具增强

#### 新增功能

- `pricingUrl` 字段: 为每个供应商添加价格链接
- 显示增强: 在供应商详情中展示价格信息
- 数据完整性: 确保所有供应商都有对应的价格链接

#### 使用示例

```bash
# 查看供应商详情（包含价格链接）
node scripts/setup-ai-providers.js --info openai

# 输出示例：

   描述: 最稳定的GPT模型，推理能力强

     ...
```

## 📝 使用指南

### 开发者使用

#### 1. 查看价格信息

```bash
# 查看特定供应商的价格信息
node scripts/setup-ai-providers.js --info openai

# 查看所有供应商列表
node scripts/setup-ai-providers.js --list
```

#### 2. 文档查询

- 打开 `docs/ai-api-providers.md`
- 找到对应的供应商
- 点击价格链接查看详细信息

### 用户使用

#### 1. 成本估算

- 在配置AI服务前，先查看各供应商价格
- 根据使用频率选择最经济的方案
- 了解不同模型的定价差异

#### 2. 供应商选择

- 比较不同供应商的价格优势
- 结合性能和价格选择最佳方案
- 关注价格更新和优惠活动
