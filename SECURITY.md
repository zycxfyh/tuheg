#### 📋 报告模板

请在安全报告中包含以下信息：

##### 🔍 漏洞详情

- **漏洞类型**: XSS、SQL注入、认证绕过等
- **严重程度**: 严重/高/中/低
- **受影响版本**: 具体的版本号或版本范围

##### 📝 复现步骤

```markdown
1. 前提条件：用户已登录系统
2. 操作步骤：
   - 访问特定页面
   - 执行特定操作
   - 观察结果
3. 预期结果：正常行为
4. 实际结果：漏洞利用成功
```

##### 🎯 影响评估

- **影响范围**: 影响的用户数量
- **数据风险**: 可能泄露的信息类型
- **系统影响**: 对服务可用性的影响

##### 💡 修复建议

- 可能的修复方案
- 临时缓解措施
- 相关安全补丁

## 🛡️ 安全最佳实践

### 🔧 开发安全

#### 📋 代码审查要求

- ✅ **强制安全审查** - 所有代码变更都需要安全审查
- ✅ **静态分析** - 使用 ESLint、SonarQube 等工具
- ✅ **依赖扫描** - 定期进行 npm audit 和 Snyk 扫描
- ✅ **安全编码标准** - 遵循 OWASP 安全编码指南

#### 🧪 安全测试

```bash
# 运行安全扫描
pnpm run security:scan

# 依赖漏洞检查
pnpm audit

# SAST (静态应用安全测试)
pnpm run sast

# DAST (动态应用安全测试)
pnpm run dast
```

### 🔐 数据保护

#### 📊 数据安全措施

- **加密存储** - 用户数据使用 AES-256 加密
- **传输安全** - 强制 HTTPS + TLS 1.3
- **日志安全** - 敏感信息自动过滤和脱敏
- **备份安全** - 加密备份和定期审计

#### 🔑 密钥管理

- **密钥轮换** - 定期自动轮换加密密钥
- **密钥存储** - 使用硬件安全模块(HSM)
- **访问控制** - 密钥访问需要多重认证

### 🚪 访问控制

#### 👤 身份验证

| 认证方式       | 适用场景   | 安全等级 |
| -------------- | ---------- | -------- |
| JWT + 刷新令牌 | API访问    | 🔒🔒🔒   |
| 多因素认证     | 管理员访问 | 🔒🔒🔒🔒 |
| OAuth 2.0      | 第三方集成 | 🔒🔒🔒   |
| SAML           | 企业集成   | 🔒🔒🔒🔒 |

#### 🔒 授权策略

- **最小权限原则** - 用户只获得必要权限
- **角色-based访问控制** - RBAC 权限模型
- **属性-based访问控制** - ABAC 细粒度控制
- **零信任架构** - 持续验证和最小化信任
